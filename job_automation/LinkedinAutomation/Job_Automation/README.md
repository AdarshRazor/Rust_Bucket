# Job Automation Pipeline

End-to-end automation: scrape LinkedIn hiring posts → clean emails with AI → send personalized outreach with resume.

---

## How It Works

```
Phase 1 — Scrape          Phase 2 — Clean (AI)        Phase 3 — Send
─────────────────         ────────────────────         ──────────────
LinkedIn feed        →    Gemma via NVIDIA NIM    →    Gmail SMTP
(Firefox + Selenium)      (Prompt.md template)         (resume attached)
        ↓                         ↓                          ↓
output/extracted_          emails/contacts.csv        contacts.csv updated
emails.csv                 (EmailAddress,               (Comments = "Sent")
                            FirstName, Comments)
```

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure credentials

Copy `.env.example` to `.env.local` and fill in every value:

```bash
cp .env.example .env.local
```

| Variable | What it is | Where to get it |
|---|---|---|
| `LINKEDIN_USERNAME` | Your LinkedIn email | Your LinkedIn account |
| `LINKEDIN_PASSWORD` | Your LinkedIn password | Your LinkedIn account |
| `EMAIL_ADDRESS` | Gmail address to send from | Your Gmail account |
| `APP_PASSWORD` | Gmail App Password (not your real password) | Google Account → Security → App Passwords |
| `NVIDIA_API_KEY` | API key for NVIDIA NIM (Gemma) | NVIDIA NIM dashboard |
| `NVIDIA_API_URL` | Endpoint URL for the Gemma deployment | NVIDIA NIM dashboard (stays the same unless you redeploy) |

> **Gmail App Password:** Go to [myaccount.google.com](https://myaccount.google.com) → Security → 2-Step Verification → App Passwords. Generate one for "Mail".

### 3. Resume

The resume PDF is already in `content/`. To update it, replace:
```
content/AdarshAnand__Full_Stack_AI_Engineer__Resume.pdf
```

## Usage

### Full pipeline (scrape → clean → send)

```bash
python main.py
```

### Common flags

| Flag | What it does | When to use |
|---|---|---|
| `--skip-scrape` | Skip Phase 1, use existing `extracted_emails.csv` | You already ran the scraper |
| `--skip-clean` | Skip Phase 2, use existing `contacts.csv` | You already cleaned or edited contacts manually |
| `--skip-send` | Skip Phase 3, don't send any emails | Dry run / just want to see the contacts |
| `--query "..."` | LinkedIn search query (default: `Hiring Full Stack`) | Different role targets |
| `--location "..."` | LinkedIn location filter (default: `India`) | City/region targeting |
| `--target N` | Max emails to save from scraper (default: `50`) | Control scrape volume |
| `--max-scrolls N` | LinkedIn feed scroll limit (default: `20`) | Deeper scrapes |
| `--batch-size N` | Emails per LLM call in Phase 2 (default: `50`) | Tune for token limits |
| `--delay N` | Seconds between emails in Phase 3 (default: `10`) | Rate limiting |

### Examples

```bash
# Scrape, clean, send — full auto
python main.py

# Already scraped, just clean + send
python main.py --skip-scrape

# Already have contacts.csv, just send
python main.py --skip-scrape --skip-clean

# Different role and location
python main.py --query "Hiring Backend Engineer" --location "India" --target 100

# Dry run — scrape and clean but don't send
python main.py --skip-send

# Deep scrape (more scrolls, more emails)
python main.py --target 200 --max-scrolls 40
```

## Phase Details

### Phase 1 — LinkedIn Scraper (`scripts/scraper.py`)

- Opens Firefox via Selenium (headful browser, not headless — avoids bot detection)
- Logs in with human-like mouse movement and keystroke timing
- Navigates to LinkedIn content search with your query + location
- Sets sort to **Latest** to get fresh hiring posts
- Scrolls the infinite feed, expanding "see more" on each post
- Extracts emails using regex from both post card text and full page source
- Saves to `output/extracted_emails.csv` with columns: `email`, `extracted_at`
- Leaves debug HTML snapshots in `logs/` if login or sort fails

**Known LinkedIn DOM quirks handled:**
- Two login forms in DOM (mobile hidden + desktop visible) — solved with `getBoundingClientRect()` viewport check
- Sort button is a `<div role="button">` not a `<button>` — targeted via `componentkey` attribute
- "Show results" after filter is an `<a>` tag, not a button — targeted via `href` containing `FACETED_SEARCH`

### Phase 2 — Email Cleaner (`scripts/cleaner.py`)

- Reads `output/extracted_emails.csv`
- Skips any email already present in `emails/contacts.csv` (deduplication)
- Batches remaining emails (default 50 per call) and sends to **Gemma 4 26B** via NVIDIA NIM
- Uses `docs/Prompt.md` as the instruction — tells the model to:
  - Remove `@gmail.com` addresses
  - Strip malformed/image entries
  - Extract first name from email (e.g. `aakash.sharma@co.com` → `Aakash`)
  - Use `Team` for generic addresses (`hr@`, `info@`, `jobs@`, etc.)
- Parses the CSV response from the model (handles markdown fences, whitespace)
- Appends new rows to `emails/contacts.csv`

**Gemma endpoint details:**
- Model: `google/gemma-4-26B-A4B-it`
- Provider: NVIDIA NIM (OpenAI-compatible API)
- Endpoint URL stored in `NVIDIA_API_URL` env var

### Phase 3 — Email Outreach (`scripts/outreach.py`)

- Reads `emails/contacts.csv`
- Skips rows where `Comments == "Sent"` (safe to restart mid-run)
- For each contact: personalizes `{{name}}` in the HTML template, attaches the resume PDF, sends via Gmail SMTP
- Writes `"Sent"` or `"Failed"` back to `Comments` column **after every single email** (crash-safe)
- Waits `--delay` seconds between sends to avoid spam flags

## Data Files

### `output/extracted_emails.csv`
Raw output from the scraper.
```
email,extracted_at
hr@techcorp.com,2026-06-21T14:30:00
aakash.sharma@startup.in,2026-06-21T14:30:00
```

### `emails/contacts.csv`
Cleaned contacts ready for outreach. Managed by Phase 2 and Phase 3.
```
EmailAddress,FirstName,Comments
hr@techcorp.com,Team,Sent
aakash.sharma@startup.in,Aakash,
```

- `Comments` is empty until Phase 3 runs
- Phase 3 writes `Sent` or `Failed` per row
- Script skips rows already marked `Sent` — safe to re-run after interruption

## Troubleshooting

**Login fails / CAPTCHA:**
- Check `logs/debug_login_failure.html` in a browser
- LinkedIn may require a verification — log in manually once, then re-run

**"NVIDIA_API_KEY not set" error:**
- Make sure `.env.local` exists (not `.env.example`) and has `NVIDIA_API_KEY=nvapi-...`

**contacts.csv has no FirstName / wrong parsing:**
- Check the raw LLM response by adding `log.debug(response_text)` in `cleaner.py`
- The model sometimes wraps output in markdown — the parser strips ` ``` ` fences automatically

**Gmail "Authentication failed":**
- You need an **App Password**, not your regular Gmail password
- 2-Step Verification must be enabled on your Google account first

**Emails going to spam:**
- Increase `--delay` (try 30–60 seconds)
- Avoid sending more than ~100/day from a single Gmail account
- Warm up the sending account with a few manual emails first
